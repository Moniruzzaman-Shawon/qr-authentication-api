# QRShield API

**QRShield — QR Authentication & Anti-Counterfeit.**

Django REST API for a one-time-use QR product-authentication platform. Each product
carries a **signed, single-use QR code**. A genuine product is **activated at point of
sale**; the first customer scan returns *genuine*, and any later scan returns
*suspicious* — surfacing counterfeits and duplicated codes. Suspicious scans trigger
email/SMS fraud alerts.

The app is **white-label**: name, logo, support email and accent colour are configurable
from the admin Settings page (or env), so one codebase ships for any client without edits.

## Tech Stack

- Python 3.10+ · Django 5/6 · Django REST Framework
- JWT auth (`djangorestframework-simplejwt`) + optional TOTP 2FA (`pyotp`)
- SQLite (dev) / PostgreSQL (prod)
- `qrcode` + Pillow (signed QR generation) · `reportlab` (print-ready label PDFs)
- Twilio (SMS) · SMTP (email)
- drf-spectacular (OpenAPI docs) · WhiteNoise · Gunicorn

## How the trust model works (two-phase activation)

Every product moves through a lifecycle, enforced by the verify endpoint:

| Status     | Meaning                                   | Customer scan result          |
|------------|-------------------------------------------|-------------------------------|
| `printed`  | Created at the factory, **not for sale**  | **Not activated** (alerted)   |
| `active`   | Activated at point of sale                | **Genuine** → `verified`      |
| `verified` | A customer has verified it once           | **Suspicious** → `flagged`    |
| `flagged`  | Scanned again after verification          | **Suspicious**                |

This defeats supply-chain pre-scanning: a code photographed before activation can
never verify as genuine. A database `UniqueConstraint` guarantees **at most one**
genuine scan per product, and verification runs inside a `select_for_update`
transaction, so concurrent scans can't both win.

QR payloads are **HMAC-signed** (`/verify/<uuid>?sig=...`); the API rejects missing or
tampered signatures, so leaked UUIDs can't be probed or forged. An admin **recall**
(deactivate) makes a product fail verification permanently.

## Named-product catalog

Admins curate a **catalog** of named products (e.g. "Engine Oil 5W-30"). QR-coded units
are generated by **selecting** a catalog entry — the name/brand are copied onto each unit
at creation, so there's no free-text typing and history stays stable if the catalog
later changes.

## Roles & data protection

Two roles (Django Groups), checked on every endpoint:

- **Admin** — full control: products, deletes, user management, branding, audit log,
  PII exports, activation, dashboards.
- **Operator** — point-of-sale staff: activate codes and read dashboards. No deletes,
  no user management. **Customer PII is masked** for operators (email masked; phone/IP/
  user-agent hidden), and bulk PII exports are blocked.

Public (no auth): only `verify`, `check`, and the read-only branding config.

## Security features

- JWT access/refresh with rotation + blacklist on logout
- **Brute-force lockout**: an account locks after `LOGIN_MAX_ATTEMPTS` failures for
  `LOGIN_LOCKOUT_MINUTES`
- **Optional TOTP two-factor** for any user (Google Authenticator / Authy / 1Password)
- Per-endpoint role permissions; PII masking for operators
- HMAC-signed QR codes; rate-limiting on verify/check/login
- **Audit log** of significant admin actions (who/what/when)
- Security headers (HSTS, secure cookies, SSL redirect) gated on `DEBUG=False`

## Installation (local dev)

```bash
cd api
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt

cp .env.example .env          # then edit; for local dev set DEBUG=True
python manage.py migrate
python manage.py seed_admin   # creates the Admin user from INITIAL_ADMIN_* env vars
python manage.py runserver
```

API at `http://localhost:8000`. Interactive docs at `http://localhost:8000/api/docs/`.

> **Note:** `DEBUG` defaults to **False**. Local development requires a `.env` with
> `DEBUG=True` and a `SECRET_KEY` (see `.env.example`).

## API Endpoints

### Auth, account & security
| Method | URL | Role | Description |
|--------|-----|------|-------------|
| POST | `/api/auth/login/` | public | `{username, password, otp?}` → `{access, refresh, user}` |
| POST | `/api/auth/refresh/` | public | Rotate access token |
| POST | `/api/auth/logout/` | auth | Blacklist a refresh token |
| GET | `/api/auth/me/` | auth | Current user + role + 2FA status |
| POST | `/api/auth/change-password/` | auth | Change own password |
| POST | `/api/auth/2fa/setup/` | auth | Begin TOTP enrolment (returns secret + otpauth URI) |
| POST | `/api/auth/2fa/enable/` | auth | Confirm a code to turn 2FA on |
| POST | `/api/auth/2fa/disable/` | auth | Disable 2FA (confirm password) |
| GET/POST | `/api/users/` | Admin | List / create users |
| GET/PATCH/DELETE | `/api/users/<id>/` | Admin | Manage a user (DELETE = deactivate) |

### Catalog & products
| Method | URL | Role | Description |
|--------|-----|------|-------------|
| GET/POST | `/api/catalog/` | Operator(read)/Admin(write) | Named-product catalog |
| GET/PUT/PATCH/DELETE | `/api/catalog/<id>/` | Operator(read)/Admin(write) | Catalog entry (DELETE keeps units) |
| GET | `/api/products/` | Admin/Operator | List (paginated) |
| POST | `/api/products/` | Admin | Create (status `printed`) |
| GET | `/api/products/<uuid>/` | Admin/Operator | Detail + scan history |
| PUT/PATCH/DELETE | `/api/products/<uuid>/` | Admin | Update / recall (`is_active`) / delete |
| POST | `/api/products/bulk-create/` | Admin | Bulk create from a catalog entry |
| POST | `/api/products/bulk-csv/` | Admin | Bulk create from CSV |
| POST | `/api/products/<uuid>/activate/` | Admin/Operator | Activate (printed → active) |
| POST | `/api/products/bulk-activate/` | Admin/Operator | Activate many `{ids:[...]}` |
| GET | `/api/products/download-qr/` | Admin/Operator | QR codes as ZIP (`?ids=`) |
| GET | `/api/products/download-qr-pdf/` | Admin/Operator | Print-ready A4 QR label sheet (`?ids=`) |
| GET | `/api/products/export/` | Admin/Operator | Inventory CSV |

### Verification (public)
| Method | URL | Description |
|--------|-----|-------------|
| POST | `/api/verify/<uuid>/?sig=` | Verify (records scan; genuine/suspicious/not_activated) |
| GET | `/api/check/<uuid>/?sig=` | Read status without recording a scan |

### Analytics, customers & admin
| Method | URL | Role | Description |
|--------|-----|------|-------------|
| GET | `/api/scans/` | Admin/Operator | Scan records (paginated, `?is_first_scan=`) |
| DELETE | `/api/scans/<id>/` | Admin | Delete a scan record |
| GET | `/api/scans/export/` | Admin | Scan history CSV (PII) |
| GET | `/api/stats/` | Admin/Operator | Dashboard totals, status breakdown, 14-day series |
| GET | `/api/fraud-alerts/` | Admin/Operator | Suspicious scans, classified (paginated) |
| GET | `/api/customers/` | Admin/Operator | Customers with aggregated scan data |
| GET/DELETE | `/api/customers/<email>/` | Operator(read)/Admin(delete) | Customer detail; delete removes their scans |
| GET | `/api/customers/export/` | Admin | Customer CSV (PII) |
| GET/PATCH | `/api/config/` | public(read)/Admin(write) | White-label branding |
| GET | `/api/audit/` | Admin | Audit log (paginated) |

## Environment variables

All configuration is via environment / `.env` — see **`.env.example`** for the full,
commented list. Key ones:

- `DEBUG`, `SECRET_KEY` (required in prod), `ALLOWED_HOSTS`, `CORS_ALLOWED_ORIGINS`
- `FRONTEND_URL` — embedded into every QR code (must be publicly reachable)
- `QR_SIGNING_KEY` — optional separate key for QR signatures
- `DB_*` / `DATABASE_URL` — Postgres (omit for SQLite)
- `JWT_ACCESS_MINUTES`, `JWT_REFRESH_DAYS`, `THROTTLE_*`
- `LOGIN_MAX_ATTEMPTS`, `LOGIN_LOCKOUT_MINUTES` — brute-force lockout
- `BRAND_APP_NAME`, `BRAND_TAGLINE`, `BRAND_COMPANY`, `BRAND_SUPPORT_EMAIL`, `BRAND_ACCENT`
  — seed the editable white-label config on first run
- `EMAIL_*`, `ALERT_RECIPIENT_EMAILS`, `TWILIO_*`, `ALERT_RECIPIENT_PHONES`
- `INITIAL_ADMIN_USERNAME/EMAIL/PASSWORD` — seeded by `manage.py seed_admin`

## White-labeling

Branding lives in a `SiteConfig` singleton, seeded from `BRAND_*` env vars on first run
and editable afterwards by an Admin under **Settings**. The SPA reads `GET /api/config/`
at startup and applies the app name (titles, logos), support email, and accent colour
across the whole UI — no code changes to rebrand for a new client.

## Notifications

On a suspicious or not-activated scan, the brand is alerted by **email** (SMTP; console
in dev) and **SMS** (Twilio, if configured). Sending is fail-safe — a provider outage
never blocks verification. Set `NOTIFY_CUSTOMER_ON_GENUINE=True` to also email customers
a verification receipt.

## Deployment

The repo root ships a `docker-compose.yml` (Postgres + gunicorn API + nginx-served
client). From the repo root:

```bash
cp api/.env.example .env   # fill in real values; set DEBUG=False and a strong SECRET_KEY
docker compose up --build
```

App at `http://localhost:8080`. The API container runs migrations and `seed_admin` on
start. Generate a production secret key with:

```bash
python -c "from django.core.management.utils import get_random_secret_key as k; print(k())"
```

### Production checklist
- [ ] `DEBUG=False`, strong 50-char `SECRET_KEY`, real `ALLOWED_HOSTS` / `CORS_ALLOWED_ORIGINS`
- [ ] `FRONTEND_URL` = the real public site (QR codes embed it)
- [ ] Postgres configured (`DB_*`), not SQLite
- [ ] TLS terminated in front of nginx; `SECURE_SSL_REDIRECT=True`
- [ ] SMTP + Twilio credentials and `ALERT_RECIPIENT_*` set
- [ ] Branding set under **Settings** (or via `BRAND_*`)
- [ ] `INITIAL_ADMIN_PASSWORD` set, then rotated after first login; enable 2FA
- [ ] Media volume backed up / pointed at object storage (S3) for multi-instance
